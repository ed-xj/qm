Motion Command Arguments

The arguments described below apply to the motion commands, MOVE, PICK, PLACE, SEIZE, RELEASE and, partially, to RETRACT. See also the entries for those specific commands.

 

Syntax

Arguments are either lone keywords, like linear, or they consist of a keyword followed by one or more numeric values. "Syntactic sugar" The processor will parse (and ignore) commas and equal signs, which the host program may use to make the commands more readable. The semicolon, " ; " can be used for comments. All symbols on one line following

a semicolon will be ignored.

 

Arguments

Station name: The first argument to any motion command must be a station name. All other arguments are optional.

 

Inside or perch, if present, must appear after the station name. Other arguments may appear in any order.

 

Inside/Perch: For motion commands other than pick and place, the second argument may be the keyword inside or perch, indicating which pose is the commanded or target pose. If neither keyword is second, perch is assumed. Pick and place do not require the inside or perch keywords, and will ignore them if present.

 

The remaining arguments, enumerated below, may occur in any order. If an argument requires one or more values, they are coded as floating point numbers which may be in any C or C++ format.

 

Index {num}: Specify which slot. Indexes begin at one. Default is one. A negative index in a motion command other than pick or place means that the motion is to finish at the low, or stroked-down position for the indicated slot. For example, move "Tool" index = -2 means to move to the stroked-down position at slot number 2. (The equal sign is optional.) A positive index means to come in at the stroked-up or high position. Pick and place ignore the sign of the index, as they automatically perform the stroke-up or stroke-down as is appropriate.

 

Offset {num} {num} {num} {num} [num]: Move to the station position plus a specified offset. Offsets are by default specified in millimeters (z) and degrees (shoulder, elbow, wrist). If the end-effector for the station has a flipper, specify a fifth number, the flipper offset, in degrees.

 

Note: The offset should be small enough that it remains safe to move between perch and inside points in a straight line.

 

Offset xyzw {num}{num}{num}{num}[num]: Move to the station position plus an offset specified as xyz Cartesian coordinates (millimeters) and wrist angle (degrees). If the end-effector for the station has a flipper, the offset must specify a fifth number, the flipper offset in degrees.

 

Note: The offset should be small enough that it remains safe to move between perch and inside points in a straight line.

 

Offset tool {num}{num}{num}{num}: Move to the station position plus an offset specified as xyz Cartesian coordinates (millimeters) and tool angle (degrees), with respect to the tool reference frame.

 

Linear: Move the end-effector center point in a straight line. By default, motions from perch to inside or inside to perch at the same station are linear. Other moves are joint interpolated by default.

 

Blend {percent}: Blend the motion with any subsequent motion. Specify a blend coefficient in the range –1 to 100. By convention, a blend of –1 means to come to a stop and seek the command position with highest accuracy. Minus one is the default that is used if there is no blend argument in the command. Other numbers indicate how much a following motion command can blend (round the corner) with the one containing the blend argument. The represents a percentage of the motion’s deceleration time, the time during which the motion would decelerate to a stop if no following motion were to blend with it.

 

Blend commands associated with pick or place commands automatically apply to the last portion of the command, i.e., the move out to the perch position, which blends into the following move.

 

WARNING: Blending may occasionally lead to undesireable effects, such as cutting corners that are not meant to be cut, or moves that exceed allowable acceleration or velocity (which will result in errors). Be cautious, and use low speeds until the safety of the robot and equipment is assured.

 

Specialized versions of the blend command apply only to parts of pick or place commands. "Pblend" controls the blending of the move from the perch to the inside positions."Iblend" controls the blending of the move from the inside position through the stroke. "Sblend" controls the blend from the stroke move through the move back out to perch.

 

Preceding the blend percentage with the letter " F " (e.g. "F100") uses the minimum of the deceleration time of the current move and the acceleration time of next move to decide when to start the next move. This improves performance when blending a long move into a following short move by completely aborbing the short move’s acceleration and deceleration profile into that of the longer move.

 

Note: Elevator-type (vertical) stations behave somewhat differently, since during pick operations they move directly to the "inside" positions. By using the "iblend" parameter with the " F " prefix, the move coming in to pick a wafer can be smoothed out. In this case, the blend absorbs the short move from index -1 to index 1into the move coming into the

initial index -1 position. Example: "pick elevator iblend F100".

 

Speed {percent}: Move with a top speed on each joint of at most the specified percentage of the joint speed limit. The argument must be in the range 1-100. The joint speed limit itself is the product the global speed setting which is a robot tuning parameter set in the factory by IRSI, the percentage specified by this SPEED parameter, and the speed override percentage, which is set on the teach pendant or using the SPEED command. In other words, the speed setting in the motion command and the speed parameter the host can set with the SPEED command are multiplicative.

 

Specialized versions of the speed command apply only to parts of pick or place commands. "Ispeed" controls the speed of the move from perch to inside. "Sspeed" controls the speed during the inside stroke move. "Ospeed" controls the speed of the move from inside to perch.

 

Acc {percent}: Move with accelerations specified as a percentage of the standard acceleration. The argument must be 20 or greater. The top limit is yet to be determined. Use extreme caution when moving at high accelerations. Test at lower accelerations, and increase the number gradually. There is no global ACC command that is analogous to the

global Speed command described above.

 

Push_x {num} Push_y {num}: – Move the end effector beyond the inside low position by the amount specified in mm, with respect to the tool reference frame, then return to the inside low position and continue with the operation. Applies to Pick operations only.

 

WARNING: The "Push" argument is intended for end effectors having a spring-loaded pusher mechanism. Use of this argument with any other end effector type may result in wafer or equipment damage.

 

Delay {num}: – On a place operation, delay moving from the inside high position to the inside low position by the amount specified in milliseconds to allow the vacuum chuck to repressurize. This is generally not needed for horizontal (e.g., cassette or chuck) stations. This is intended for vertical stations where the wafer or other payload is not as well supported. Applies to Place operations using a vacuum chuck end effector only.
