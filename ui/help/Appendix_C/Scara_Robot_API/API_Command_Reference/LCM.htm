LCM - Last Complete Motion; report last complete motion.

type: immediate

Syntax

lcm

Arguments

None.

Description

Sends the host information describing the last motion the robot successfully completed. The current implementation does not remember motions that happened prior to the last time the controller was powered up.

 

The host will receive a number of status lines. Each begins with $S followed by the lcm command text as the host sent it, followed by a key word and information.

 

Key words and info:

CMD – The text of the host’s motion command that precipitated the motion.

MSG –The event message that the controller sent back to the host at the completion of the motion.

STATION – The name of the station that the motion command referred to.

TYPE – The type of motion used to reach the position – either JOINT or LINEAR.

POSE – The pose-identifier, either INSIDE or PERCH.

INDEX – The index of the commanded motion (Negative numbers indicate the stroked-down position at the given index.).

CMD-Z – The commanded Z position in millimeters.

CMD-S – The commanded shoulder angle in degrees.

CMD-E – The commanded elbow angle in degrees.

CMD-W – The commanded wrist angle in degrees.

CMD-F – If the end-effector is on a flipper, the commanded flipper angle in degrees.

 

The CMD information represents the actual position, after all indexing, stroke adjustment, and offsets (if any) are applied.

 

See also: RTP, RCP.

Response

[Robot status lines]...

Events

!E END-LCM<tab>{host command, as received}

Examples

Example 1:

lcm

>

!S lcm CMD <None>

!E END-LCMlcm

 

The host sent the command lcm before the robot had completed any motion after power-up.

 

Example 2:

12pickem pick pdoone index 5

12pickem>

!E PERCH-PICK12pickem pick pdoone index 5;

!E INSIDE-PICK12pickem pick pdoone index 5;

!E STROKE-PICK12pickem pick pdoone index 5;

!E TEACH-ON

 

The host sent the command "12pickem pick pdoone index 5". Notice that there is a cookie, "12pickem" attached to the command.

 

The robot proceeded to the perch position at station pdoone, then inside, and then completed the stroke from index 5 low (index –5) to index 5 high. Someone turned on the teach pendant, stopping the robot’s motion.

 

Now the host sends the lcm command, and receives the responses.

lcm

>

!S lcm CMD 12pickem pick pdoone index 5

!S lcm MSG STROKE-PICK

!S lcm STATION "pdoone"

!S lcm TYPE JOINT

!S lcm POSE INSIDE

!S lcm INDEX 5

!S lcm CMD-Z 41.503

!S lcm CMD-S 276.602

!S lcm CMD-E 325.104

!S lcm CMD-W -5.124

!S lcm CMD-F -0.147

!E END-LCMlcm

 

Notice that the first response (CMD) contains the host’s command as it was received, complete with the identifying cookie.

 

The next response (MSG) shows the event message that the controller sent back to the host when the motion completed. We see that the message indicated the completion of the "stroke" operation in the pick command.

 

The STATION response indicates that the motion involved a station named "pdoone".

 

The TYPE response indicates that the motion was a joint-interpolated motion, as opposed to a linear motion. (When only the Z stage is commanded to move, the motion type is JOINT, despite the fact that the motion is necessarily a linear one.)

 

The POSE response shows that the motion was relative to the inside taught position for the station.

 

The INDEX response indicates that the motion went to the high position at index 5. (If it had gone to the low position, the index number would be –5 rather than 5.)

The remaining fields are the actual commanded Z travel and joint angles.
