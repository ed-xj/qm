Set - Set a station database attribute.

type: queued

Syntax

set {station} {inside | perch} {Z | *} {{shoulder | *} {elbow | *} {wrist | *} {flip | *}

set (station) {inside | perch} near {station2} xyzwpr {x} {y} {z} {w} {p} {r}

set {station} {pitch | stroke | slots | efnum} {num}

set {stations} type {cassette | prealigner [{1 | 2}]| elevator}

set rtheta {true | false}

set (station) mapcal {LeftLow, LeftHigh, RightHigh, RightLow}

set (station) {inside | perch} near (station2) frame (framename) xyzwpr {x} {y} {z} {w} {p} {r}

set {station} inputs {approach} {inside} {wafer}

Arguments

station: A contiguous string of alpha-numeric characters, or any printable text enclosed in double quotation marks.

 

pitch: Indicates the vertical or Z distance between slots. Must be followed by a floating point number in C or C++ format. Pitch is measured in millimeters.

 

stroke: Indicates the distance to stroke up when seizing a wafer or stroke down when releasing a wafer. Must be followed by a floating point number in C or C++ format. Stroke is measured in millimeters.

 

slots: Indicates the number of slots or indices in the station. It must be followed by a positive integer.

 

efnum: Indicates the end-effector number applicable for the station. Currently must be the integer 1 or 2.

 

cassette: Indicates the station contains horizontally oriented payloads.

 

prealigner: Indicatest the station is a prealigner. By using the optional "1" or "2" arguments, a particular prealigner may be associated with a station name. (See Align.)

 

elevator: Indicates the station contains vertically oriented payloads. "Stroke" then refers to the horizontal travel required during a pick or place operation. "Perch" and "inside" positions refer to the high and low z-positions, respectively.

 

inside or perch: Indicates that the numbers which follow are joint angles for the inside or perch pose of the station. The numbers, in C or C++ floating point format, must occur in the order Z, shoulder, elbow, wrist, and (is a flipper/gripper is present) flip. Z is measured in millimeters. The others are measured in degrees. The indicated Z position should be the high or stroked-up elevation for index 1.

 

mapCal: Indicates that the four numbers following are the Z axis positions at the station corresponding to the four corners of a mapping operation. Typically one initially sets the numbers, not with a SET command, but by executing a MAP (station) LEARN command. That has the effect of learning the four corners and recording them in the station database as though you had executed SET (station) mapCal. The command SET (station) mapCal itself is typically used only in conjunction with the SAVE, RESTORE, and GETSTATIONS commands. The SAVE command writes SET (station) mapcal instructions (among others) to the station database file. The GETSTATIONS command sends them to the host. The RESTORE command executes the commands that are in named station database file that was saved using the SAVE command. The Z axis positions are in the order, left-bottom, left-top, right-top, right-bottom.

 

* (Splat) May be given instead of a numerical argument in place of Z, shoulder, elbow, wrist or flip to leave the corresponding value unchanged. This option is intended to simplify data entry and reduce errors when only one value change is to be entered. See examples.

 

near: If the keyword near is used, the closest arm joint rotational angles to those of station2 will be adopted, thus preventing the visual ambiguity that may arise form the feature that joint angles may have values greater than 360 degrees. In other words, using the near syntax prevents the unwanted possibility of having the robot move from a station with, say, a wrist angle of 18 degrees to a station whose wrist angle, though appearing identical by visual inspection, is 360+18 = 378 degrees, causing the wrist to rotate unnecessarily druing this movement.

 

xyzwpr: This is used in conjunction with NEAR to indicate that the parameters are in XYZWPR format, rather than joint angles and displacements.

 

frame: This indicates that the XYZWPR input parameters are to be interpreted in the context of a user-defined frame, not world frame.

 

inputs: Sets the input lines to be checked before performing certain operations during MOVEs and PICK and PLACE operations. The command takes three arguments:

SET (station) INPUTS (approach), (inside), (release)

 

The three arguments respectively define the input required to approach the station, move into the station, and release the wafer.

 

Each argument must be either the keyword NONE, indicating no input checking is to be performed (the default), an asterisk or "splat" (*), indicating the value is to remain unchanged, or an attribute-value pair that specifies an input line number and the required state:

(input-number) = (state)

 

The valid values for (state) are ON and OFF. The input line specified must be physically present or an error will be generated.

 

Please refer to the Appendix B, "Using Automatic Station input-State Checking" for a full description of the SET INPUTS command.

Description

This sets one or more station attributes in the current station database. The first argument is a station name. If it contains whitespace characters, it must be enclosed in double quotation marks. After the station name comes any of several attribute names, as described above. Most attributes require one or more floating point numbers. Numbers may be in any format you can generate with a C "printf" command. Inside and perch require four numbers, with an optional fifth for a flipper angle.

Events

!E END-SET<tab>{host command, as received}

Examples

set alfa pitch 10

>

!E END-SET set alfa pitch 10

 

set alfa inside 11.11 22.22 33.33 44.44

>

!E END-SET set alfa inside 11.11 22.22 33.33 44.44

 

set alfa inside * * 77.7777 *

>

!E END-SET set alfa inside * * 77.7777 *

 

getstations

>

Set "alfa" Pitch 10.000

Set "alfa" Stroke 3.125

Set "alfa" Index 1

{...}

Set "alfa" Inside 11.110 22.220 77.7777 44.440

Set "alfa" Perch {...}

!E END-GETSTATION getstations
