Map - Perform a wafer-mapping operation on a station.

type: queued

Syntax

map {station} (data) (motion-arguments)

Arguments

The data keyword indicates that all edge data is to be returned as well as the wafer map.

See MOTION COMMAND ARGUMENTS, Appendix A.

Description

Map performs a high-level mapping operation of a station of type cassette. Two edge collection passes are made, one starting at the lower left of the cassette (facing into the cassette opening) and moving to the upper left, and another moving from the upper right to the lower right. (The exact position of these vertical moves depend upon taught station positions.) The edge data is processed, and a text string is returned that indicates wafer presence or wafer misloading in each slot of the cassette. The string is made up of numbers, one number for each slot, starting at 1. A "0" indicates no wafer seen, a "1" means a wafer is present and correctly slotted, and a "2" means a wafer was seen but is not correctly positioned.

Events

!E PERCH-MAP<tab>{host command, as received}

!E LEFT-LO-MAP<tab>{host command, as received}

!E LEFT-HI-MAP<tab>{host command, as received}

!E RIGHT-HI-MAP<tab>{host command, as received}

!E RIGHT-LO-MAP<tab>{host command, as received}

!S MAP [wafer map]

!E END-MAP<tab>{host command, as received}

 

If the data keyword was specified:

!E PERCH-MAP<tab>{host command, as received}

!E LEFT-LO-MAP<tab>{host command, as received}

!E LEFT-HI-MAP<tab>{host command, as received}

!S MAP A NumSamples=[n], NumSeen=[n]

!S MAP A [sample-num], [edge], [Z-pos]

…

(rest of left data points)

…

!E RIGHT-HI-MAP<tab>{host command, as received}

!E RIGHT-LO-MAP<tab>{host command, as received}

!S MAP A NumSamples=[n], NumSeen=[n]

!S MAP A [sample-num], [edge], [Z-pos]

…

(rest of right data points)

…

!S MAP [wafer map]

!E END-MAP<tab>{host command, as received}

 

Where:

[sample-num] = edge number

[edge] = 1 (wafer edge) or 0 (no wafer)

[z-pos] = robot height (in millimeters)
