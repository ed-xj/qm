Grip - Open or close the gripper (or vacuum) on a specified end-effector.

type: queued

Syntax

grip {1 | 2} {on | off}

Arguments

The first argument is the end-effector number, currently restricted to 1 or 2.

on: either turn on the vacuum or engage the gripper.

off: either turn off the vacuum or disengage the gripper.

Description

This command operates the vacuum or gripper on the indicated end-effector. It does not effect any robot arm or flipper motion.

 

If the end-effector is a gripper, it is an error to attempt a grip command while any motions are in progress or pending in the queue.

 

Gripper Retry. In the case of a gripping end-effector, if a gripper operation should fail (either grip or ungrip), the operation will be retried once. If the second operation succeeds, a normal event message is returned. If the second operation fails, a fail event message is returned. An internal count of retries is maintained, which is accessible using the status command.

 

See also: Seize, Release, Pick, Place, Status.

Events

!E GRIP-RETRY-GRIP<tab>{host command, as received}

!E END-GRIP<tab>{host command, as received}

Examples

2134 Grip 1 On

2134>

!E END-GRIP2134 Grip 1 On
