Learn - Set a station pose to the current position and end-effector number.

type: queued

Syntax

learn {station} {inside | perch} (index)

Arguments

The first argument is a station name. If it contains whitespace characters, it must be enclosed in double quotation marks. After the station name comes either the keyword inside or perch.

 

inside: Indicates that the pose being learned is inside the named station, as opposed to the perch pose, which is supposed to be a safe approach point for the station.

 

perch: Indicates that the position is the perch, or safe approach pose for the named station.

 

index: see Appendix A, MOTION COMMAND ARGUMENTS.

Description

Sets the configuration for a station, either the inside position or the perch position, as indicated in the parameters. The current robot position (and flipper position if the end-effector has a flipper) is used to set the station data.

 

If no index number is specified as a third argument, he learned robot position will correspond to the high, or stroked up, Z elevation for index (slot) 1 of the pose. See Appendix A, MOTION COMMAND ARGUMENTS, Pick, and Place for information on indexing.

 

The purpose of the perch position is to indicate a pose from which it is safe to go into the station (the inside position) with the end-effector traveling in a straight line. Currently, it is the userâ€™s responsibility to see to it that, that is the case.

 

The controller will make all motions between the inside and perch positions linear, unless the station has the RTHETA attribute set to true, in which case the move will be done using joint-interpolation.

 

Note: The "current end-effector" is used to set the station. See command EFnum.

 

It is an error to attempt to learn a position when the robot is in motion.
