Config - Access system configuration settings.

type: queued

Syntax

config {save|restore} {filename}

config get (category (item))

config set {category} {item} (=) {value}

Arguments / Description

save {filename}: Saves configuration data to the specified file. If no filename is specified, configuration data is saved to the default configuration file.

 

restore {filename}: Configuration data is first read from the file on the controller, and is used to reinitialize any corresponding devices.

 

get: With no trailing argument, this call displays all configuration settings. The verbosity can be reduced by specifying config get (category), config get (category) (item), or using the splat (*) symbol followed by an arbitrarily long list of literals, each of which will then be displayed. Example: config get * (com_Port | com_baud | simulate).

 

set: Updates the configuration with new values. An equal sign between (item) and (value) is optional and may be omitted.

 

A complete list of configuration variables is included in Appendix C, Configuration Options.

Events

!E END-CONFIG<tab>{host command, as received}
