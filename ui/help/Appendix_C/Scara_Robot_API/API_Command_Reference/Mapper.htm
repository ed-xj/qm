Mapper - Operate the wafer mapper.

type: queued

Syntax

mapper {init | on | off | status}

mapper data [XYZWPR]

mapper data frame (frame-name)

Arguments

init: Indicates the mapper should be initialized for a new MAP operation. This clears map edge data.

on: Turns the mapper laser on, and enables the capture of edge data.

off: Turns the mapper laser off, and disables edge capture.

status: Returns the state of the mapper.

data: Returns all edge data seen since the last MAPPER INIT. The XYZWPR keyword indicates the edge data should be returned as full coordinate data instead of only Z position.

frame: If present, specifies that the coordinates (data) that the controller sends to the host should be relative to the named user-defined frame, not the robot’s world frame.

Description

Mapper commands allow fine control over mapping, by allowing the user to select when the mapper is on, what robot moves occur during that time, and when the data is retrieved. Note, however, that only raw map data is returned, so the user must perform their own calculations on the data to derive wafer presence, cross-slotting, etc.

 

MAPPER INIT clears the capture data memory, initializes any mapper hardware, and turns the mapper off. Data will not be captures in this state.

 

MAPPER ON turns on the mapper illuminator and enables edge capture. MAPPER OFF turns off edge capture, but leaves the system in a state ready to perform further edge capture. Multiple MAPPER ON/MAPPER OFF calls may be done after a MAPPER INIT to disable edge capture when the robot is moving past objects that could generate spurious edges.

 

When the DATA argument is given alone, all edge data captured since the first MAPPER ON command after a MAPPER INIT call is returned. The first line of the returned data will have the format of:

!S MAP A NumSamples=[n], NumSeen=[n]

 

Two numbers are returned since the capture buffer is finite, and there may have been more edges seen than could fit into the capture buffer.

 

The above line will then be followed by {TotalCaptures} number of lines, each of which will have the format:

!S MAPPER DATA {Num}, {Dir}, {Z}

 

where {Num} is the number of the edge, starting with one. {Dir} is the direction of the edge: "1" represents detection of an object, "0" the loss of an object. In general, if the first edge has a direction of "0", then the start of the object was missed; if two edges in a row have the same direction, then an edge was dropped during edge capture and should be considered an error. Exceptions to these cases are when the user has knowingly started the mapper on an obstruction, or has used the MAPPER ON/OFF commands to control capture when positioned on an obstruction. {Z} specifies the Z axis position where the edge was seen.

 

When the optional XYZWPR argument is used with the DATA argument, each of the DATA lines returns the X, Y and Z positions in millimeters, as well as yaw or wafer rotation around the Z axis in degrees (W), pitch (P, always zero), and roll or flipper rotation around the end-effector X axis in degrees (R). This data represents the end effector’s "wafer center point" location and orientation in world space:

!S MAPPER DATA {Num}, {Dir}, {X}, {Y}, {Z}, {W}, {P}, {R}

 

This allows the user to perform more complex operations such as mapping wafers in horizontal cassettes, and determination of reference points in the robot space ("autoteach").
