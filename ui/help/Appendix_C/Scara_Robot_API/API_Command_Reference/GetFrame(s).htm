GetFrame(s) - Output listing of user-defined frames.

type: immediate

Syntax

getframe(s) (frame-name)

Arguments

frame-name: Single name of user-defined frame to limit output (optional).

Description

The command returns to the host text that may later be sent back to the controller to reestablish the user-defined frames as it was when the getframes command was originally sent from the host. Because the controller output is intended to be used later as commands to the controller, the output does not follow the standard conventions. Each line of text that the controller returns begins with the keyword setFrame That keyword uniquely identifies the response line as a reply from the getframes command.

 

The END-GETFRAMES event will not be posted until after all the setframe commands have been sent.

 

If you issue the command getframes without a parameter, it returns information for the entire user-frames database. If you supply a frame ] name as a parameter, it returns only information for that one frame.

 

See also: Set, Save, Restore.

Response

[Setframe commands] …

Events

!E END-GETFRAMES<tab>{host command, as received}

Examples

29getfr getframes

29getfr>

SetFrame "my frame" 34.45 –55.3, 55.562, -90

...

!E END-GETFRAMES29getfr getframes
