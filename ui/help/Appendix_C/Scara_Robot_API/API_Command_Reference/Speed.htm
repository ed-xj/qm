Speed - Set speed-overide setting.

type: queued

Syntax

speed (percent)

Arguments

An integer between 1 and 100 inclusive, indicating the percentage of the rated maximum speed at which motions are to be executed.

Description

The effect of this command is identical to that of changing the speed setting on the teach pendant.

 

The effect of this command is multiplicative with the effect of the speed setting on individual motion commands. Thus if the speed-override is set to 60 percent, and a motion command is qualified with a speed of 50 percent, then the motion will execute at 30 percent of rated maximum speed.

 

Note: Changing the speed-override setting will change the path taken during the "blend" or overlapped portions of blended motions. At higher speeds, the path will round the corners more than at low speeds.

Events

!E END-SPEED<tab>{host command, as received}

Examples

Move station_one

Speed 50

Robot goes to 50 percent of rated speed almost immediately.

 

Move station_one

Wait modone

Speed 50

Move station_two

Robot goes to 50 percent of rated speed after first motion completes.
