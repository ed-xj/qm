Move - Move to a specified station pose (inside or perch).

type: queued

Syntax

move {station} (motion-arguments)

Arguments

See MOTION COMMAND ARGUMENTS, Appendix A.

Description

This command moves the robot from one pose to another.

 

Joint-interpolated motion: If the keyword linear is not present in the motion command, the joint motions are interpolated between the start and target positions. Long motions use a position profile that is piecewise cubic with a continuous second derivative (acceleration). In the robotic literature, a position profile of that type is often called an Scurve.

 

The acceleration profile is thus a continuous piecewise-linear function. Very short motions use a first order profile. The standard maximum jerk, maximum acceleration, and maximum speed on each joint are set by IRSI at the factory. Those can be overridden, within bounds, using the global speed setting (speed command), and the speed and acc keywords in the motion command itself. See Appendix A, MOTION COMMAND ARGUMENTS.

 

Linear motion: If the keyword linear is present in the motion command, the tool center point of the end-effector will travel in a straight line. In that case, the tool center point acceleration profile along the straight line will be a continuous piecewise-linear function for long motions.

 

The end of one motion may be blended with the beginning of another by using the blend keyword and a value on the first command. See Appendix A, MOTION COMMAND ARGUMENTS.

 

It is an error to change indexes (slots) while inside a station.

 

It is an error to attempt to move to the inside position of a station to anywhere other than the perch position of the same station.

Events

!E END-MOVE<tab>{host command, as received}

Example 1

Example 2

The following commands show how a "via" point can be indexed, just like a station that applies to a physical object like a tool or load port. If no further motion commands follow soon after the third one given here, the robot will come to a smooth stop at the "load 1" station perch point outside slot 8. If another motion command follows shortly, the robot will not exactly reach the perch point, but will instead blend the next motion with the last one give here.

 

321 move tool perch, index -4, blend 50

322 move "via 1" index 8, blend 100

323 move "load 1" index 12, blend 75
