Wait - Wait for completion of an action.

type: queued

Syntax

wait {align | modone | flipper | prealigner [{1 | 2}]}

wait input {1 | 2 | 3 | 4 | 5 | 6 | 7 | 8} {off | on}

Arguments

align: waits until a prealigner alignment operation has completed.

 

modone: waits until all queued robot motions have completed. DOES NOT WAIT FOR FLIPPER MOVES.

 

flipper: waits until a flipper motion has completed.

 

input: waits until the digital input line specified by {input-num} is at the specified state. Valid input line numbers are 1 to 8 in the base system; optional I/O may increase the available lines.

 

prealigner: waits until the default prealigner is finished. Using the optional arguments "1" or "2" waits for the specified prealigner to finish.

Description

Wait causes the command queue to pause until the specified action is complete or condition met. Wait only effects the execution of queued commands; the command interpreter will accept commands, immediate mode commands will be executed, and queued commands will still be appended to the queue, even if a wait is active.

Events

!E END-WAIT-ALIGN<tab>{host command, as received}

!E END-WAIT-MODONE<tab>{host command, as received}

!E END-WAIT-INPUT<tab>{host command, as received}

Examples

move prealigner ; go to the aligner perch

wait align ; done yet?

pick prealigner ; get the aligned wafer

 

The robot will move to the "prealigner" station, and decelerate to a stop. When the alignment operation is complete, it will then proceed with the pick operation, removing the wafer from the "prealigner" station.

 

move load2 blend 70 ; move to loadport 2

wait input 2 on ; wait for the door to open

place load2 speed 60 ; put wafer in loadport

 

The robot will move to the "load2" station, and if input 2 is not on when the robot reaches the deceleration point for the station, it will decelerate to a stop at the perch point for "load2". Since the move command before the wait has a blend of 70, if input 2 was already on, the robot would only decelerate slightly before continuing to the inside position of "load2" to do the wafer place. If input 2 should become on after the robot has reached the deceleration point (and is decelerating to a stop) the robot will then resume the move.

 

move "via 1" ; move robot clear

wait modone ; wait until it's there

output 1 on ; close the door

 

The robot will move to the "via 1" station and decelerate to a stop. As soon as the robot has stopped, output 1 will be set to 1. Without the wait modone command, output 1 would be set on as soon as the move to "via 1" had started.
