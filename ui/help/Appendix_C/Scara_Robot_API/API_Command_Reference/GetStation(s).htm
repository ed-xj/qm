GetStation(s) - Output listing of station data.

type: immediate

Syntax

getstation(s) (station-name)

Arguments

station-name: Single station name to limit output (optional).

Description

Returns to the host text that can later be sent back to the controller to reestablish the station database as it was when the getstations command was originally sent from the host. Because the controller output is intended to be used later as commands to the controller, the output does not follow the standard conventions. Each line of text that the controller returns begins with the keyword set. That keyword uniquely identifies the response line as a reply from the getstations command.

 

The END-GETSTATIONS event will not be posted until after all the set commands have been sent.

 

If you issue the command getstations without a parameter, it returns information for the entire station database. If you supply a station name as a parameter, it returns only information for that one station.

 

See also: Set, Save, Restore.

Response

[Set commands] …

Events

!E END-GETSTATIONS<tab>{host command, as received}

Examples

29getst getstations

29getst>

Set "Home" Pitch 6. ; Notice there is no cookie

Set "Home" Slots 1

Set "Home" EFNum 1

Set "Home" Inside 0.000, 0.000, 180.000, 0.000

Set "Home" Perch 0.000, 0.000, 180.000, 0.000

Set "Load 1" Pitch 6.350
