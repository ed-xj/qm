RTP - Read Target Position; return the commanded (target) position.

type: immediate

Syntax

rtp

Arguments

None.

Description

Sends the host the target position of the robot, in joint angle space. The target position is the position the robot will reach if all outstanding motion commands complete successfully. If the robot has been commanded to at least one position, and there are no motion commands outstanding, the target position is the same as the current position. Status

lines begin with !S (see example below). The first status line returned will indicate the station, the next the pose (inside or perch), followed by the index or slot number. A positive slot number means the high position for the slot; A negative number means the low position. The next four lines will give the joint angles of the target position.

 

See also: RCP.

Response

{Robot status lines}...

Events
