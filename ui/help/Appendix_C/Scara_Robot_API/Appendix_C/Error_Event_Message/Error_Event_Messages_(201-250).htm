Error Event Messages (201-250)

Number 201
Message Configuration changed. Recycle power.
Applies to Configuration
Recovery Cycle power
Action Configuration changes require power to be cycled to become effective.

 

Number 203
Message Mapper type can be NONE | WX43 | FX301.
Applies to Configuration
Recovery Reset
Action Verify that the command you are using is syntactically correct.

 

Number 204
Message Unable to load/save configuration.
Applies to Configuration
Recovery Reset
Action Verify flash card on SBC property seated. If error persists, replace flash card.

 

Number 205
Message Continuity check failed.
Applies to Motion commands
Recovery Reset
Action If error recurs, set "HWMODONES OFF" and "HWFLIP OFF" until controller can be replaced.

 

Number 207
Message Argument can be {All | Error_state}.
Applies to Logfile
Recovery Reset
Action Correct command.

 

Number 208
Message Wafer present in the move path.
Applies to Motion commands
Recovery Reset
Action Check wafer presence on end-effector and prealigner and send a safe move.

 

Number 209
Message Syntax: INIT PREALIGNER (COMx) (baud).
Applies to Init
Recovery Reset
Action Correct command.

 

Number 210
Message Unable to communicate with Robot(LGB).
Applies to Operation
Recovery Reset
Action Check that grey cable from controller to RMU is properly connected.

 

Number 211
Message Maximum of 9 user0defined frames supported.
Applies to Setframe
Recovery Reset
Action Delete a user frame (Deleteframe) and retry command.

 

Number 212
Message Frame does not exist.
Applies to Setframe, RCP, mapper, getframe
Recovery Reset
Action Supply a previously define user frame name.

 

Number 213
Message Mapping Data was not initialized.
Applies to Map
Recovery Reset
Action Re-issue command.

 

Number 215
Message Not all of the required stations have been learned in order to map.
Applies to Map
Recovery Reset
Action Learn the required mapping stations.

 

Number 216
Message Redundant motion command argument(s).
Applies to Motion commands
Recovery Reset
Action Verify that the command you are using is syntactically correct.

Recovery Reset
