Error Handling

As commands are parsed, a syntax error may be encountered. When a syntax error occurs, the device will return the question mark character and set the error status. This error status must be cleared with the ESC command before any motion commands can be executed. If a command parses correctly but fails execution, it will return the character F. One possible cause for a motion command to return F is that the error status was not clear, when the command was issued. The ESL command can be used to read the error code of the last error encountered. The ESF command will return the error code of the first error encountered after the error status was last cleared.

 

Refer to the end of this document for a list of all the error codes.
