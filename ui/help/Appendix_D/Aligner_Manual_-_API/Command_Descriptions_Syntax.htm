Command Descriptions Syntax

The commands are all case sensitive.

 

In the description of the commands that follow, the # character signifies a place where the host must send a single literal character such as 0 or 3. The command description will specify the allowable range of values. This is independent of the binary parameter option described below.

 

Some commands take or return a numeric value, which may not be specified in one digit. These commands signify a placeholder for the parameter with <u16bit>, <u32bit>, <s16bit>, or <s32bit>. The format of these parameters and return values will vary dependent on the binary parameter option described below.
