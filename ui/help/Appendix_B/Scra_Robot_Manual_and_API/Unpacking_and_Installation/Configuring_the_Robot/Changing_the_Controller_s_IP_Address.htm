Changing the Controller's IP Address

The controller comes from the factory with a default IP address of 10.0.99.99, and a subnet mask of 255.255.255.0. If this is not compatible with your application, the IP address and subnet mask can be changed. This instruction describes how to change the IP address of the IRSI controller/robot in order to be compatible with different networks.

 

Materials required:

  • Host Computer

  • Null Ethernet Cable

 

Procedure:

The IP address of host computer must be temporarily changed to communicate with IRSI controller/robot. This will be IP: 10.0.X.X (where "X.X" is some address other than the controler's) and subnet mask: 255.255.0.0. These instructions assume you are using Windows 2000.

  1. On the host computer, right click on Network Places.

  2. Click on Local Area Connection.

  3. Click properties.

  4. Highlight Internet Protocol (TCP/IP).

  5. Click properties. (Make a note of the addresses you change so you can change them back when you are finished.)

  6. Change the IP address to : 10.0.X.X.

  7. Change the subnet mask should be: 255.255.0.0.

  8. Click "OK".

 

The next step is to connect to the controller using FTP, and retrieve the tcpip.cfg file from the controller.

  1. Connect a null cable from host computer to IRSI controller port labeled "Ethernet".

  2. From the host's Start menu, click "Run...".

  3. In the "Open:" text box, type "cmd", and click on "OK".

  4. In the DOS console window, at the command prompt, type "cd\" and press "Enter" to go to the computer's root directory.

  5. At the command prompt, type "md irtemp" to create a temporary directory where you can place and edit file.

  6. At the command prompt, type "cd irtemp" to enter the newly created temporary directory.

  7. At command prompt type "FTP 10.0.99.99".

  8. When prompted for a user ID, press "Enter".

  9. When prompted for a password, press "Enter".

  10. At the command prompt, type "get tcpip.cfg".

 

The next step is to edit the tcpip.cfg file

  1. Using windows Explorer, navigate to the "irtemp" directory.

  2. Open the tcpip.cfg with a text editor. The second line of the file (the first IP address) is the IP address assigned to the controller. Change this to the desired IP address. The third line of the file (the second address) is the subnet mask. Change this to the desired subnet mask value.

  3. Save the changes to the file, and exit the text editor.

  4. At command prompt in the console window, type "put tcpip.cfg".

  5. End the FTP session by typing "bye" in the console window, followed by "exit".

  6. Restore the host computer's original TCP/IP configuration using the notes from above.

  7. Power cycle the IRSI controller.
