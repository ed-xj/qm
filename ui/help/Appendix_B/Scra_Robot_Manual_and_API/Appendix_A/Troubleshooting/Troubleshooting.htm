Troubleshooting

This appendix covers the steps necessary to troubleshoot the robot and  controller. This section is organized by problem type, in the order of likely occurrence, i.e., startup first.

 

Trouble shooting is presented as a series of possible symptoms. With each symptom are one or more items to check. These are followed by one or more possible remedies, with the most likely listed first. Start with the first remedy, and go through the list until the problem is solved.

 

In general, the sequence of troubleshooting is as follow:

  • Check the software configuration, using the "config get" command. Most errors are the result of incorrect configurations. The M820 API manual has a detailed appendix covering configuration settings.

  • Check the connections. Are all connections secure and correct, including vacuum? Are pre-aligners (if any) plugged into the correct connector?

  • Replace cables. Cable failures are rare. However, since they are exposed, they are susceptible to externally-induced damaged.

  • Replace suspect in this appendix deals with popped belts, which may occur if the robot is driven into some obstacle.
