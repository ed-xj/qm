End Effector Issue

Symptom: Vacuum end effector is installed, and the command "grip 1 on" succeeds, but no vacuum is present on the end effector.

 

Check: Issue the command "config get robot". Verify that "sim_com" is set to "false"

Remedy: Issue the following commands:

config set Robot Sim_Com = false

reset

If the problem is solved, issue the command:

config save

 

Check: Issue the command "config get endeffector1" or "config get endeffector2", depending on the end effector concerned. Verify that "SimGripper" and "Sim_Grip_Detect" are set to "false".

Remedy: Issue the following commands

config set EndEffector1 SimGripper = false

config set EndEffector1 Sim_Grip_Detect = false

config set EndEffector2 SimGripper = false ; if used

config set EndEffector2 Sim_Grip_Detect = false ; if used

reset

If the problem is solved, issue the command:

config save

 

Check: Issue the command "config get robot". Verify that "VacSwap" is set to "false". This setting allows the user to switch the activation of the vacuum to the wrist vacuum barbs if necessary. If set to true, "grip 1 on" would activate vacuum barb 2 and vice versa.

Remedy: Issue the following commands:

config set Robot VacSwap = false

reset

If the problem is solved, issue the command:

config save

 

 

Symptom: Vacuum end effector is installed, but issuing the command "grip 1 on" results in a "vacuum sense timeout error".

 

Check: Issue the command "config get endeffector1" or "config get endeffector2", depending on the end effector concerned. Verify that "Type" is set to "VacGrip".

Remedy: Issue the following commands:

config set EndEffector1 Type = VacGrip

reset

If the problem is solved, issue the command:

config save

 

Check: Issue the command "config get robot". Verify that "VacSwap" is set to "false". This setting allows the user to switch the activation of the vacuum to the wrist vacuum barbs if necessary. If set to true, "grip 1 on" would activate vacuum barb 2 and vice versa.

Remedy: Issue the following commands

config set Robot VacSwap = false

reset

If the problem is solved, issue the command:

config save
