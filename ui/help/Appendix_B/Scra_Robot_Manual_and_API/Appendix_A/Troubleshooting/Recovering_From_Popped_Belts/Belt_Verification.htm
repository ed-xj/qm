Belt Verification

When the robot arm is in a configuration with the rotary joints (shoulder, elbow, wrist) at 0, 180, and 0 degrees, the arm should be folded up on itself, with the aftarm (top) directly over the fore-arm (bottom) and the end-effector directly over the fore-arm, as in the following picture:

 

Figure 1 - Joint Reference Position

 

You can use the teach pendant to jog the robot to the correct position (press the "Pos" key to display the joint angles), but a more accurate way is to create a "home" station and move the robot to it. Be sure before executing the move that the robot can move to this position without obstruction.

 

The commands to send from the host are:

  1. Set Home Perch 0, 0, 180, 0

  2. Move Home

 

(Note: the exact direction the arms are pointing may be different than the above example depending upon the world frame setting.) If the robot arm is not in this configuration, then there likely has been a belt-pop or a joint slip.

 

Belt-pop usually occurs in only one joint, so two of the three joints should be in agreement. Either the aft- and fore-arm segments are lined up and the end-effector is skewed (in which case the wrist has moved), or one of the arm segments is parallel to the end-effector and another is not (in which case the non-parallel joint has moved).
