Power-up, Connection Troubleshooting

Symptom: Power switch on, no sound coming from controller.

 

Check: Power supply cable is connected to controller.

Remedy: Connect power supply cable to controller.

 

Check: Power supply cable is connected to proper voltage supply.

Remedy: Connect power supply cable to supply voltage.

 

Check: Power intake module is set to correct supply voltage.

Remedy: Set power intake module voltage to correct voltage (see chapter 6).

 

 

Symptom: Power switch on, fan noise coming from controller, but no power light.

 

Check: Plug in teach pendant, or try to connect remotely.

Remedy: If teach pendant initializes, or connection succeeds, suspect failed power light.

Remedy: If teach pendant does not initialize, or connection fails, suspect controller single-board computer or flash RAM failure. Return to vendor for repair.

 

 

Symptom: Power switch on, fan noise coming from controller, but teach pendant display blank.

 

Check: Teach pendant is plugged into the controller.

Check: Teach pendant displayed self test information during startup. Suspect controller single-board computer or flash RAM failure. Remedy: Return controller to vendor for repair.

Check: Teach pendant did not display self test information during startup. Suspect teach pendant failure. Remedy: Return teach pendant to vendor for replacement.

 

 

Symptom: Teach pendant displays random characters.

 

Remedy: Teach pendant is not configured correctly. Return to vendor for replacement.

 

 

Symptom: Power switch on, fan noise coming from controller, but controller doesn't respond to connect requests.

 

Check: Using correct connection cable. If using ethernet with a direct connection to the controller, cable must be of crossover type. If using serial connection, cable must be of null modem type.

Remedy: Use correct connection cable.

 

Check: Use teach pendant options menu to determine controller's IP address and subnet mask configuration.

Remedy: Re-configure host to use this configuration. After connecting successfully, modify controllerâ€™s tcpip.cfg file to configure controller addresses as needed.

Remedy: Suspect controller single-board computer or flash RAM failure. Return to vendor for repair.

 

 

Symptom: Upon powering up and issuing the "reset" command, received the following error messages:

!E ERROR 51 - Prealigner-Unit-1 : Prealigner not responding to

init: Error initializing prealigner

!E ERROR 100 - reset: Reset command failed. Possible hardware

failure

!E ERROR 271 - reset: Abort called

!E ERROR 4 - reset: First command after fault, abort, master

or power-up must be RESET

 

Check: Prealigner not being used in this application. Issue the commands "config

get prealigner1" and "config get prealigner2".

Remedy: Issue the following commands:

config set Prealigner1 Sim = True

config set Prealigner1 Sim = True

config save

reset

 

Check: Issue the commands "config get prealigner1" and "config get

prealigner2". The results should look similar to the following:

config set Prealigner1 Sim = True

config set Prealigner1 Type = PA800

config set Prealigner1 Com_Port = COM9

config set Prealigner1 Com_Baud = 19200

config set Prealigner1 Wait_Place = False

config set Prealigner1 Calc = True

config set Prealigner1 Hand_Off = Chuck

config set Prealigner1 Align_Mode = Prealigner

config set Prealigner1 Theta_Adjust = 180.000

Remedy: Correct any incorrect entries. See the M820 API manual for further details.

 

Check: Prealigner used in this application. Verify vacuum source and connections between prealigner(s) and controller.

Remedy: Correct/attach vacuum source. Issue the following commands" reset

init prealigner 1 com9 19200

init prealigner 2 com10 19200 ; if used

Remedy: Correct cable connections between prealigner(s) and controller. Issue the commands listed above.

Remedy: Replace prealigner cables with spare units and issue the commands listed above. If successful, return non-working cables to vendor for repair.

Remedy: Replace prealigners with spare units and issue the commands listed above. If successful, return non-working prealigner to vendor for repair.

Remedy: Replace controller with spare unit and issue the commands listed above. If successful, return non-working controller to vendor for repair.
