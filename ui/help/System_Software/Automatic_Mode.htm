Automatic Mode

Automatic mode will transfer materials from a selectable input to a selectable output. In order to use the automatic mode you must ensure the following criteria has been met. The recovered, the system must contain a valid recipe, a valid material map must be present. The user interface supports all three of these operations. Under the system tab you will find the following sections.

 

  1. System Level Buttons

  • Load Recipe - Used to load selected recipe.

  • Start Recipe - Used to start the automatic recipe.

  • Stop Recipe - Used to stop the automatic recipe.

  • Error Recover - Used to recover all robotics and clear all errors.

  • Speed Slow - Used to select the automation transfer speed.

  • Speed Medium - Used to select the automation transfer speed.

  • Speed Fast - Used to select the automation transfer speed.

 

  1. Tool Layout - Used to show current layout of tool.

 

  1. Sub-component Status - Used to show error status of sub components.

 

  1. Logging Level - Used to adjust the debug logging level of the tool.

 

  1. Home Tool - Used to Home entire tool upon power up, or starting the software.



While using automatic mode the system software will run a selectable recipe that can be configured by the user. The RECIPE tab displays the current configuration and options available to the user. It is here that you can build and run user defined recipes.

 

  1. Select Station Input and Available Slots - Used to select input station.

    • Select the input station to be used in the recipe.

    • Select the valid slots to transfer wafers from the input station.

    • Select the unload order. Either begin unloading from the bottom up or the top down.

 

  1. Select Station Output and Available Slots - Used to select output station.

    • Select the output station to be used in the recipe.

    • Select the valid slots to transfer wafers to the output station.

    • Select the load order. Either begin loading from the bottom up or the top down.

 

  1. PROCESS 1 - Configure process 1 with available commands.

    • If enabled, process 1 commands will execute in sequential order.

    • See Appendix A. for details on valid commands.

 

  1. PROCESS 2 - Configure process 2 with available commands.

    • If enabled, process 2 commands will execute in conjunction with process 1 commands.

    • See Appendix A. for details on valid commands.


Pre Process and Post Process

 

Each recipe can be configured to have pre and post process operations. Pre process operations will take place prior to the transfer of any material. Post process operations will take place after the last device has been transferred to its output station.

 

  1. Pre Process - Configure pre process with available commands.

    • If enabled, pre process commands will execute in sequential order.

    • See Appendix A. for details on valid commands.

 

  1. Post Process - Configure post process with available commands.

    • If enabled, post process commands will execute in sequential order.

    • See Appendix A. for details on valid commands.
