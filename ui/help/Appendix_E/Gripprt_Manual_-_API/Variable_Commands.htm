Variable Commands

VR<name>

Reads the value of the variable named in the argument <name> from RAM. The return format depends on the variable read.

VFR<name>

Reads the value of the variable named in the argument <name> from RAM like the VR command, but also lists the bits if the variable is a bitfield type variable.

VBS<name><bit>

Sets the bit number specified in the <bit> param of the bitfield variable specified in the <name> param. Bits are numbered with the LSB as zero.

VBC<name><bit>

Clears the bit number specified in the <bit> param of the bitfield variable specified in the <name> param. Bits are numbered with the LSB as zero.

VW<name><val>

Writes the value specified in the second argument <val> to the variable named in the first argument <name> to RAM. The argument value depends on the variable being set.

VL

Outputs a list of all the variables available and their current values. This command is intended for manual use on a terminal for diagnostics. One line is output for each variable and uses the syntax of the VW command to allow simple cut and paste operations for setting new values.

VFL

Lists all variables as the VL command, but also lists the bits for each bitfield type variable.

 

The following variables are available:

comm_cnfg <u32bit> Serial host communications configuration bits.
can_comm_cnfg <u32bit> CANBUS host communications configuration bits.
sys_cnfg <u16bit> System configuration bits.
io_cnfg <u16bit> System I/O configuration bits.
level_cnfg <u16bit> System I/O levels configuration bits.
type <u16bit> Gripper type identifier returned by GT command.
baud <u32bit> Baud rate for host serial communications.
jog_vel <u16bit> Slow velocity for jogging linear travel of gripping tips.
pin_jog_vel <u16bit> Slow velocity for jogging pin rotational movement of gripping tips.
grip_vel <u16bit> Velocity used during grip motion.
grip_acc <u16bit> Acceleration used during grip motion.
release_vel <u16bit> Velocity used during release motion.
release_acc <u16bit> Acceleration used during release motion.
*extend_vel <u16bit> Velocity used during extend motion of special type gripper.
*extend_acc <u16bit> Acceleration used during extend motion of special type gripper.
*retract_vel <u16bit> Velocity used during retract motion of special type gripper.
*retract_acc <u16bit> Acceleration used during retract motion of special type gripper.
pin_vel <u16bit> Velocity used during pin rotation of gripping tips.
pin_acc <u16bit> Acceleration used during pin rotation of gripping tips.
pin_down_psn <u16bit> Allows software adjustment of rational tips in the down state.
*grip_psn <u16bit> Pincher gripped position for special type gripper.
grip_fail_psn <u16bit> Max travel of gripping tip from released position.
*extend_fail_psn <u16bit> Max travel of gripping tip from released position.
grip_force <u16bit> Grip holding force.
deglitch_filter <u16bit> Discrete input sampling filter count for deglitching signals.
move_current_0 <u16bit> Current setting for stepper 0 motor while in motion.
hold_current_0 <u16bit> Current setting for stepper 0 motor while in motion.
move_current_1 <u16bit> Current setting for stepper 1 motor while in motion.
hold_current_1 <u16bit> Current setting for stepper 1 motor while in motion.
move_current_2 <u16bit> Current setting for stepper 2 motor while in motion.
hold_current_2 <u16bit> Current setting for stepper 2 motor while in motion.

 

* These variables are not used by most grippers but are used on clip applicator style grippers only.

 

Note: Editing the variables above should only be done with full comprehension of what the effects. This document does not provide the information needed for such understanding. Please contact QM to learn more.
